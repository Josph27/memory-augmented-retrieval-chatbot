# src/agents/ — Agent Wrappers

**TL;DR:** Six agent wrappers form the orchestration layer. Only `CoordinatorAgent`
(540 lines) calls other agents — it owns the full turn pipeline: route → save message →
retrieve → rerank → build context packet → [optional LangGraph] → generate answer →
save assistant message → memory update. `ContextManagerAgent` (541 lines) is the
deterministic context builder (budget allocation → evidence selection → packet
assembly). The remaining four are thin adapters: `ChatAgent` (18 lines, model wrapper),
`ShortTermMemoryAgent` (25 lines, short-term context), `ContextBuilderAgent` (38 lines,
legacy context path), and `DocumentIngestionAgent` (75 lines, Chroma indexing).

---

## Agent roles and responsibility boundaries

| Agent | Lines | Role | Calls other agents? |
|---|---|---|---|
| `CoordinatorAgent` | 540 | Turn orchestrator. Owns the full pipeline from routing to answer. | Yes — all others |
| `ContextManagerAgent` | 541 | Deterministic budget + evidence selection + packet construction | No |
| `ContextBuilderAgent` | 38 | Legacy context builder. Wraps `ShortTermMemory.build_model_messages()`. | No |
| `ChatAgent` | 18 | Thin wrapper around `ModelWrapper.chat()`. | No |
| `ShortTermMemoryAgent` | 25 | Adapter for `ShortTermMemory`. Loads context and triggers memory updates. | No |
| `DocumentIngestionAgent` | 75 | Loads and indexes local document files into Chroma. | No |

Only `CoordinatorAgent` calls other agents. All other agents are self-contained
wrappers around deterministic service classes. The `CoordinatorAgent` instantiates
its own `ContextManagerAgent` inside `run_turn()` when one is not injected —
making it the only agent that constructs another agent at runtime.

## CoordinatorAgent: full turn pipeline

`CoordinatorAgent.run_turn()` runs the complete turn as a sequential pipeline:

```text
1. Routing
   routing_agent.route(content) → RoutingDecision → RoutePlan
   (If task_context="document_qa": force document_memory in route_plan)

2. Persist user message
   database.save_message(chat_id, role="user", content) → user_message_id

3. Retrieval
   retriever_dispatcher.retrieve(chat_id, route_plan) → MemoryCandidate[]

4. Reranking
   memory_reranker.rank_with_trace(candidates, ranking_profile, query) → ranked MemoryCandidate[]

5. Context packet (new path — authoritative)
   context_manager.build_context_packet(
       system_prompt, latest_user_message, ranked_candidates, route_plan
   ) → ContextManagerResult(context_budget, context_packet, metadata)

6. [Optional] LangGraph shadow/demo
   If orchestration_mode ≠ "native":
     run_read_only_langgraph_orchestration() → OrchestrationResult
     If mode == "langgraph_demo": replace authoritative result with LangGraph output.
     On LangGraph error: record error, fallback to native.
     compare_orchestration(native, langgraph) for shadow comparisons.

7. [Still runs] Legacy context building (comparison only)
   memory_agent.build_context() → ShortTermContext
   context_builder.build(chat_id, system_prompt, context, latest_user_message) → model_messages
   context_comparator.compare(old_messages, new_context_packet) → ContextComparison

8. Prompt assembly
   context_packet_to_model_messages(packet, latest_user_message, context_comparison)
   → PromptAssembly(messages[], valid, fallback_reason)
   If invalid: fallback to legacy model_messages from step 7.

9. Answer generation
   chat_agent.generate(final_model_messages) → answer string
   Error handling:
     - OpenAIError / TimeoutError: graceful error message, turn saved for retry
     - LangChainChromaUnavailable: retrieval-unavailable message
     - LangGraph demo + retrieval errors: skip LLM call, return error message
     - LangGraph demo + insufficient evidence: skip LLM call, return evidence message

10. Persist assistant message
    database.save_message(chat_id, role="assistant", content=answer) → assistant_message_id

11. Memory update
    memory_agent.update_memory_if_needed(chat_id) → bool
    Errors caught silently — memory updates never break the visible chat response.

12. Trace
    WorkflowTrace assembled with all metadata, timings, candidate counts.
    _log_trace() emits compact console output.
    Return AgentTurnResult(answer, chat_id, trace_id, termination_reason, trace, …)
```

### Error handling philosophy

Errors are caught and recorded, not propagated:

- **Retrieval errors**: appended to `errors[]`, included in trace metadata.
- **LangGraph errors**: recorded as `orchestration_error`; native result used as
  fallback. In `langgraph_demo` mode, `orchestration_fallback_used=True`.
- **Answer generation errors**: `answer_failed=True`, graceful error message
  returned to user, turn is retryable.
- **Memory update errors**: caught silently, appended to `errors[]`. The next
  successful turn can retry because unprocessed messages remain.

### Timing instrumentation

Every stage is timed with `perf_counter()` and recorded in `timings{}`:

| Key | Stage |
|---|---|
| `route_planning` | Routing decision |
| `save_user_message` | DB persistence |
| `retrieval` | Dispatcher.retrieve() |
| `reranking` | Reranker.rank_with_trace() |
| `context_budget_allocation` | ContextManagerAgent (includes selection) |
| `context_packet_building` | Same as above (combined timing) |
| `context_packet_validation` | PromptAssembly validation |
| `legacy_context_building` | Legacy ContextBuilderAgent.build() |
| `context_comparison` | ContextComparator.compare() |
| `langgraph_orchestration` | LangGraph shadow/demo run (when enabled) |
| `main_model_call` | LLM chat generation |
| `save_assistant_message` | DB persistence |
| `update_memory_if_needed` | Memory update scheduling |
| `total_turn` | Full turn wall-clock time |

### Trace output

`_log_trace()` emits two compact console lines per turn:

```text
# Decision trace
workflow_trace trace_id=<uuid> chat_id=<id> intent=<intent> active_sources=[...]
  routing_fallback=<bool> routing_reason=<reason> reranker_mode=<mode>
  retrieved_candidates=<N> ranked_candidates=<N> context_profile=<profile>
  estimated_prompt_tokens=<N> overflow_detected=<bool> prompt_source=<source>
  termination_reason=<reason>

# Timing trace
turn_timing trace_id=<uuid> route_planning_ms=<N> retrieval_ms=<N>
  reranking_ms=<N> context_budget_allocation_ms=<N> main_model_call_ms=<N>
  total_turn_ms=<N>
```

## ContextManagerAgent: the authoritative context builder

`ContextManagerAgent` replaced the legacy `ContextBuilderAgent` as the primary
context construction path. It does everything deterministically — no LLM calls.

### When it's used

`CoordinatorAgent` always runs the new path (step 5). The legacy path (step 7)
still runs for comparison logging but is not used for answer generation unless
the new path's prompt assembly fails validation.

### Construction

Two construction paths:

1. **Direct injection** (manual constructor): all dependencies passed explicitly.
2. **`for_model()` factory** (recommended): auto-configures from model identity.

```python
agent = ContextManagerAgent.for_model(
    model_id="gemma-3-27b-it",
    endpoint_context_window=131072,
    processor_loader=...,
    ...
)
```

`for_model()` resolves the model profile, resolves the effective context window,
builds a token estimator (HF Gemma tokenizer), and wires everything together.

### `build_context_packet()` pipeline

```text
ranked_candidates, route_plan, system_prompt, latest_user_message
  │
  ▼
1. Budget allocation
   budget_allocator.allocate(route_plan, candidates, model_context_limit,
                              answer_reserve, system_prompt)
   → ContextBudget with per-source token splits

2. Compute available memory budget
   hard_input_budget = max_tokens - output_reserve - safety_margin
   Subtract: system_tokens, query_tokens, chat_template_overhead
   → available_memory_budget

3. Required-evidence preflight
   selector.select(candidates, route_plan, token_budget=unbounded, ...)
   Extract required floor candidates (required_raw_evidence,
   required_scope_evidence, protected_recent_suffix)
   → required_floor token cost

4. Working memory budget plan
   budget_planner.plan(route_plan, available_memory_budget, required_evidence_floor)
   → BudgetPlan with working_memory_budget
   If available < required floor: budget expanded to cover required evidence.
   Context profile caps: memory_recall → max 8K, document_question → 32K,
   global_summary → up to 128K.

5. Evidence-constrained selection
   selector.select(candidates, route_plan, token_budget=working_memory_budget, ...)
   → SelectionResult(selected_candidates[], dropped_candidates[],
                      evidence_contract_satisfied, token_usage)

6. Context packet assembly
   context_builder.build(system_prompt, latest_user_message, ranked_candidates,
                          context_budget, route_plan, preselected_candidates=…)
   → ContextPacket with model-ready messages, token accounting

7. Overflow trimming
   If final_prompt_tokens > hard_input_budget:
     Remove lowest-utility non-required candidates one at a time.
     If still over: remove lowest-rank required candidate.
     If still over: mark evidence_contract_satisfied=False, add
     "hard_input_budget" to missing_requirements.
   Rebuild packet after each removal until budget fits.

→ ContextManagerResult(context_budget, context_packet, metadata)
```

### Key design decisions

- **Source budgets are advisory only.** `source_token_budgets` are computed but
  not enforced — evidence requirements can override them.
- **Required evidence is budget-protected.** If the available memory budget can't
  cover required evidence, the budget is expanded. The answer reserve or safety
  margin may be consumed.
- **Selection is requirement-first.** `EvidenceConstrainedContextSelector`
  satisfies evidence requirements first (raw spans, structured memory, document
  citations, recent messages), then fills remaining budget with optional
  candidates ordered by utility.
- **Overflow is handled online.** If the assembled packet exceeds the hard input
  budget, candidates are trimmed iteratively in a while-loop.

## Legacy vs. new context building

Two context building paths coexist in every turn. Both run, but only one
produces the prompt used for answer generation.

### Legacy path (`ContextBuilderAgent`)

```text
ShortTermMemoryAgent.build_context(chat_id)
  → loads recent messages, structured memory state, gist if available
  → ShortTermContext(raw_messages[], memory_state, gist)

ContextBuilderAgent.build(chat_id, system_prompt, context, latest_user_message)
  → ShortTermMemory.build_model_messages(system_prompt, context, latest_user_message)
  → formats structured memory into prompt text via format_memory_for_prompt()
  → builds ContextPacket with recent_message_ids and model_messages
  → returns (model_messages, ContextPacket)
```

**Limitations:**

- No per-source budget allocation — unstructured "pour everything in"
- No evidence requirement enforcement — no contract between routing and context
- No overflow trimming — can exceed context window silently
- No candidate-level provenance in the packet
- No token accounting per candidate

### New path (`ContextManagerAgent`)

See full pipeline above. Advantages:

- Per-source budget allocation with advisory splits
- Working memory budget planning with profile caps
- Evidence-constrained selection (required evidence gets budget priority)
- Iterative overflow trimming if budget exceeded
- Full token accounting per candidate and per prompt section
- Evidence contract satisfaction tracking

### Comparison and fallback

`ContextComparator` compares both paths and records warnings (message count
differences, token discrepancies). The comparison result is attached to the
new path's prompt assembly for trace purposes.

If the new path's `PromptAssembly.valid` is `False` (e.g., message validation
failure), the system falls back to the legacy model messages:

```python
prompt_source = "context_packet"  # normal
# or
prompt_source = "legacy_short_term_memory_fallback"  # fallback
```

Fallback is rare — it triggers only on structural message list validation
failures, not on budget or evidence issues.

## Agent coordination

`CoordinatorAgent` is the only agent that calls other agents. All coordination
is sequential and explicit — there is no event bus, message queue, or shared state
between agents.

### Dependency injection

All agents follow constructor injection:

```python
coordinator = CoordinatorAgent(
    database=db,
    memory_agent=ShortTermMemoryAgent(short_term_memory),
    context_builder=ContextBuilderAgent(short_term_memory),
    chat_agent=ChatAgent(ModelWrapper(model_name)),
    system_prompt="You are a helpful assistant...",
    route_planner=RoutePlanner(),
    retriever_dispatcher=RetrieverDispatcher(db),
    memory_reranker=MemoryReranker(),
    context_budget_allocator=ContextBudgetAllocator(),
    trace_context_builder=TraceContextBuilder(),
    context_comparator=ContextComparator(),
    routing_agent=RoutingAgent(route_planner),
    context_manager_agent=ContextManagerAgent.for_model(model_id),
)
```

Any unset optional dependency gets a sensible default inside `__init__()` —
the coordinator is self-assembling when dependencies are omitted.

### Runtime agent construction

`CoordinatorAgent` constructs a `ContextManagerAgent` at the start of each turn
if one was not injected:

```python
context_manager = self.context_manager_agent or ContextManagerAgent(
    budget_allocator=self.context_budget_allocator,
    context_builder=self.trace_context_builder,
)
```

This means the `context_manager_agent` field is optional — if injected, it's reused
across turns; if omitted, a new one is created each turn with shared allocator/builder.

### Agent interaction summary

```text
CoordinatorAgent
  │
  ├──→ RoutingAgent.route(content) → RoutingDecision
  ├──→ RetrieverDispatcher.retrieve(chat_id, route_plan) → MemoryCandidate[]
  ├──→ MemoryReranker.rank_with_trace(candidates, …) → ranked MemoryCandidate[]
  ├──→ ContextManagerAgent.build_context_packet(…) → ContextManagerResult
  │      ├── [uses] ContextBudgetAllocator
  │      ├── [uses] EvidenceConstrainedContextSelector
  │      ├── [uses] DynamicWorkingMemoryBudgetPlanner
  │      └── [uses] ContextBuilder
  ├──→ ShortTermMemoryAgent.build_context(chat_id) → ShortTermContext
  ├──→ ContextBuilderAgent.build(chat_id, system_prompt, context, …) → (messages, packet)
  ├──→ ContextComparator.compare(old, new, …) → ContextComparison
  ├──→ ChatAgent.generate(messages) → answer
  ├──→ ShortTermMemoryAgent.update_memory_if_needed(chat_id) → bool
  └──→ [optional] LangGraph read-only shadow/demo orchestration
```

## ChatAgent

Thin wrapper. Exists as an explicit agent boundary for answer generation:

```python
class ChatAgent:
    def generate(self, messages: list[dict[str, str]], temperature=None) -> str:
        return self.model.chat(messages, temperature=temperature)
```

No logic beyond the delegation call. The `ModelWrapper` handles API communication,
retry logic, and response parsing.

---

## ShortTermMemoryAgent

Adapter for `ShortTermMemory`. Two methods:

- `build_context(chat_id, latest_user_message_id)` → loads recent messages,
  structured memory state, and gist (if available) into a `ShortTermContext`.
- `update_memory_if_needed(chat_id)` → runs the structured memory update policy
  (LangMem extraction, deduplication, upsert). Returns `True` if any memories
  were created or updated.

---

## ContextBuilderAgent (legacy)

Wraps `ShortTermMemory.build_model_messages()` to produce model-ready messages from
the legacy short-term memory path. Notable details:

- Formats structured memory into prompt text only when `memory_state_is_not_empty()`.
- Produces a `ContextPacket` with `recent_message_ids` populated from raw messages.
- The `model_messages` list includes system prompt, formatted memory, message
  history, and the latest user message in LLM-ready `[role, content]` format.

---

## DocumentIngestionAgent

Loads and indexes local files into Chroma for document RAG:

```python
def index_file(path, display_name=None, document_id=None) -> DocumentIngestionResult
```

Returns `DocumentIngestionResult(document_id, file_name, file_extension,
chunk_count, indexed, errors, metadata)`.

Uses the deterministic document loading and indexing pipeline from
`src/documents/loaders.py` (not an LLM). Falls back to
`LangChainChromaRetriever.from_env()` when no indexer is injected.

## Configuration

| Env variable | Default | Effect |
|---|---|---|
| `ORCHESTRATION_MODE` | `"native"` | `"native"`, `"langgraph_shadow"`, or `"langgraph_demo"`. Controls whether LangGraph pipeline runs. |
| `SEMANTIC_ROUTER_ENABLED` | unset | When truthy, `CoordinatorAgent` uses `SemanticRouter`. |
| `ROUTING_MODE` | `"rule"` | `"rule"`, `"llm"`, or `"hybrid"`. |
| `MODEL_NAME` | required | Model ID used by `ChatAgent` and `ContextManagerAgent.for_model()`. |
| `CHAT_SYSTEM_PROMPT` | see `config.py` | The system prompt injected into `CoordinatorAgent`. |
