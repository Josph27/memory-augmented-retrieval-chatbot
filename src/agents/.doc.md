# src/agents/ — Agent Wrappers

**TL;DR:** Six agent wrappers form the orchestration layer. Only `CoordinatorAgent`
(741 lines) calls other agents — it owns the full turn pipeline: route → save message →
retrieve → rerank → neighbor expansion → build context packet → [optional LangGraph] →
retrieval log → legacy context → compare → generate answer → save assistant message →
memory update. `ContextManagerAgent` (541 lines) is the deterministic context builder
(budget allocation → evidence selection → packet assembly → overflow trimming).
The remaining four are thin adapters: `ChatAgent` (18 lines, model wrapper),
`ShortTermMemoryAgent` (25 lines, short-term context), `ContextBuilderAgent` (38 lines,
legacy context path), and `DocumentIngestionAgent` (129 lines, Chroma indexing + LLM
document summarization).

---

## Agent roles and responsibility boundaries

| Agent | Lines | Role | Calls other agents? |
|---|---|---|---|
| `CoordinatorAgent` | 741 | Turn orchestrator. Full pipeline: route → retrieve → expand → rerank → context → answer → memory update. | Yes — all others |
| `ContextManagerAgent` | 541 | Deterministic budget + evidence selection + packet construction + overflow trimming. | No |
| `ContextBuilderAgent` | 38 | Legacy context builder. Wraps `ShortTermMemory.build_model_messages()`. | No |
| `ChatAgent` | 18 | Thin wrapper around `ModelWrapper.chat()`. | No |
| `ShortTermMemoryAgent` | 25 | Adapter for `ShortTermMemory`. Loads context and triggers memory updates. | No |
| `DocumentIngestionAgent` | 129 | Loads, indexes files into Chroma, and generates LLM document summaries. | No |

Only `CoordinatorAgent` calls other agents. All other agents are self-contained
wrappers around deterministic service classes. The `CoordinatorAgent` instantiates
its own `ContextManagerAgent` inside `run_turn()` when one is not injected —
making it the only agent that constructs another agent at runtime.

## CoordinatorAgent: full turn pipeline

`CoordinatorAgent.run_turn()` runs the complete turn as a sequential pipeline:

```text
1. Routing
   routing_agent.route(content) → RoutingDecision → RoutePlan
   (If task_context="document_qa": force document_memory via require_document_memory())
   (If chat has associated documents: force via require_chat_document_memory(),
    set force_enabled_sources={"document_memory"})

2. Persist user message
   database.save_message(chat_id, role="user", content) → user_message_id

3. Retrieval
   retriever_dispatcher.retrieve(chat_id, route_plan) → MemoryCandidate[]

4. Reranking
   memory_reranker.rank_with_trace(candidates, ranking_profile, query)
   → ranked MemoryCandidate[]

5. Neighbor expansion (NEW)
   retriever_dispatcher.expand_document_neighbors(ranked_candidates)
   → expanded candidates
   Runs AFTER reranking so the cross-encoder saw clean single-chunk text.

6. Context packet (new path — authoritative)
   context_manager.build_context_packet(
       system_prompt, latest_user_message, ranked_candidates, route_plan
   ) → ContextManagerResult

7. Retrieval log dump (NEW)
   If RETRIEVAL_LOG_ENABLED: compute turn_index, dump all ranked candidates to
   logs/retrieval/<chat_id>/turn_<N>.json with rank, scores, window flag, in_prompt.

8. [Optional] LangGraph shadow/demo
   If orchestration_mode ≠ "native":
     run_read_only_langgraph_orchestration(
         ..., task_context=task_context,
         force_enabled_sources=force_enabled_sources)
     → OrchestrationResult
   On LangGraph error: record error, fallback to native.

9. [Still runs] Legacy context building (comparison only)
   memory_agent.build_context() → ShortTermContext
   context_builder.build(...) → model_messages
   context_comparator.compare(...) → ContextComparison

10. Prompt assembly
    context_packet_to_model_messages(packet, latest_user_message, context_comparison)
    → PromptAssembly(messages[], valid, fallback_reason)
    If invalid: fallback to legacy model_messages.

11. Answer generation
    chat_agent.generate(final_model_messages) → answer
    Error handling:
      - InferenceServerUnreachable: graceful error, turn saved for retry
      - OpenAIError / TimeoutError: graceful error, turn saved for retry
      - LangChainChromaUnavailable: retrieval-unavailable message
      - LangGraph demo + retrieval errors: skip LLM, return error message
      - LangGraph demo + insufficient evidence: skip LLM, return evidence message

12. Persist assistant message
    database.save_message(chat_id, role="assistant", content=answer)
    → assistant_message_id

13. Memory update
    memory_agent.update_memory_if_needed(chat_id) → bool
    Errors (incl. InferenceServerUnreachable) caught silently.

14. Trace
    WorkflowTrace with full metadata, timings, saved/retrieved memory rows,
    document rows, orchestration metadata, retrieval_errors, turn_index.
    Return AgentTurnResult(answer, chat_id, trace_id, termination_reason, trace, …)
```

### Two document-scope functions

- **`require_document_memory()`** — same-turn attachment. Overrides intent to
  `document_question`, sets context_profile to `document_question`, adds
  `same_turn_attachment: True` filter.
- **`require_chat_document_memory()`** — sticky document scope. Enables
  `document_memory` but does NOT override intent or context_profile. Controlled by
  `CHAT_DOCUMENT_SCOPE_STICKY` env var (default `true`).

### Error handling philosophy

- **Retrieval errors**: appended to `errors[]`, included in trace.
- **LangGraph errors**: recorded as `orchestration_error`; native fallback used.
- **Answer generation errors**: `answer_failed=True`, retryable error message.
- **Memory update errors**: caught silently, appended to `errors[]`.

### Timing instrumentation

| Key | Stage |
|---|---|
| `route_planning` | Routing decision |
| `save_user_message` | DB persistence |
| `retrieval` | Dispatcher.retrieve() |
| `reranking` | Reranker.rank_with_trace() |
| `neighbor_expansion` | expand_document_neighbors() |
| `context_budget_allocation` | ContextManagerAgent (full pipeline) |
| `context_packet_building` | Same as above (combined) |
| `context_packet_validation` | PromptAssembly validation |
| `legacy_context_building` | Legacy ContextBuilderAgent.build() |
| `context_comparison` | ContextComparator.compare() |
| `langgraph_orchestration` | LangGraph shadow/demo run |
| `main_model_call` | LLM chat generation |
| `save_assistant_message` | DB persistence |
| `update_memory_if_needed` | Memory update scheduling |
| `total_turn` | Full turn wall-clock time |

### Retrieval log dump

When `RETRIEVAL_LOG_ENABLED=True` (default), writes all ranked candidates to
`logs/retrieval/<chat_id>/turn_<N>.json`. Each row: rank, file_name, chunk_id,
similarity, raw_ce, z_norm_ce, deterministic, final_score, window, in_prompt,
mode, source, content (≤500 chars). Deduplicated via `_candidate_log_id()`.

Module-level helpers: `_candidate_log_id()`, `_compact_breakdown()`,
`_dump_retrieval_log()`, `require_document_memory()`,
`require_chat_document_memory()`, `elapsed_ms()`, `bounded_error()`.

### Trace output

`_log_trace()` emits two console lines enriched with:
context_comparison_warnings, token_estimator, overflow_detected, overflow_tokens,
recent_message_ids, reranker_fallback.

### Trace metadata

Trace carries: `saved_memory_rows`, `retrieved_memory_rows`,
`retrieved_document_rows`, `retrieval_errors`, `turn_index`, `answer_status`,
and full `orchestration` metadata including `task_context` and `force_enabled_sources`.

## ContextManagerAgent: the authoritative context builder

`ContextManagerAgent` does everything deterministically — no LLM calls.

### Construction

Two paths:

1. **Direct injection**: all dependencies passed explicitly.
2. **`for_model()` factory** (recommended): auto-configures from model identity.

```python
agent = ContextManagerAgent.for_model(
    model_id="gemma-3-27b-it",
    endpoint_context_window=131072,
    processor_loader=...,
)
```

Accepts optional `minimum_optional_candidate_utility` and
`raw_span_overlap_threshold`.

### `build_context_packet()` pipeline

```text
ranked_candidates, route_plan, system_prompt, latest_user_message
  │
  ▼
1. Budget allocation
   budget_allocator.allocate(...) → ContextBudget with per-source token splits

2. Compute hard input budget
   hard_input_budget = max_tokens - output_reserve - safety_margin
   Subtract: system_tokens, query_tokens, chat_template_overhead
   → available_memory_budget

3. Required-evidence preflight
   selector.select(candidates, route_plan, token_budget=2^62, ...)
   Extract floor via required_floor_candidates():
     required_raw_evidence, required_scope_evidence, protected_recent_suffix
   For LONG_DOCUMENT_TASK_CONTEXTS: also include all document_memory candidates.
   Measure floor cost via required_floor_token_cost().

4. Working memory budget plan
   budget_planner.plan(route_plan, available_memory_budget, required_evidence_floor)
   → BudgetPlan with working_memory_budget
   Caps: memory_recall→8K, document_question→32K, global_summary→up to 128K.

5. Evidence-constrained selection
   selector.select(candidates, route_plan, token_budget=working_memory_budget, ...)
   → SelectionResult

6. Context packet assembly
   context_builder.build(...) → ContextPacket

7. Overflow trimming (while-loop)
   If final_prompt_tokens > hard_input_budget:
     removable_overflow_candidate() → remove lowest-utility non-recent optional
     (if none: oldest recent message)
     required_overflow_candidate() → remove highest-rank required (last resort)
     If none: mark evidence_contract_satisfied=False
   Rebuild packet each iteration.

→ ContextManagerResult(context_budget, context_packet, metadata)
```

### Module-level helpers

| Helper | Purpose |
|---|---|
| `required_floor_candidates(preflight, route_plan)` | Extracts required candidates + all doc_memory for long-doc task contexts. |
| `required_floor_token_cost(...)` | Builds preview packet to measure floor cost. |
| `removable_overflow_candidate(selection)` | Lowest-utility non-recent optional, or oldest recent message. |
| `required_overflow_candidate(selection)` | Highest-rank required (last resort removal). |
| `recent_message_order(candidate)` | Orders by source_message_ids min or record_id. |
| `context_manager_metadata(...)` | Builds trace metadata: source budgets, evidence selection, included/dropped counts by source, token accounting. |
| `candidate_counts_by_source(candidates)` | Selected candidate counts per source. |
| `dropped_counts_by_source(dropped)` | Dropped candidate counts per source from metadata. |

### Key design decisions

- **Source budgets are advisory only.** Evidence requirements can override them.
- **Required evidence is budget-protected.** Budget expands if floor can't fit.
- **Selection is requirement-first.** Required evidence first, then optional by utility.
- **Overflow handled iteratively.** While-loop trimming with sorted removal order.
- **Long-document task contexts.** All document_memory candidates included in required floor.

## Legacy vs. new context building

Both paths run every turn. Only one produces the prompt used for generation.

### Legacy path (`ContextBuilderAgent`)

Loads recent messages, structured memory, gist → formats memory as system message
→ builds ContextPacket. Limitations: no budgets, no evidence enforcement, no
overflow trimming, no candidate provenance, no per-candidate token accounting.

### New path (`ContextManagerAgent`)

Per-source budgets, working memory planning, evidence-constrained selection,
iterative overflow trimming, full token accounting, evidence contract tracking.

### Fallback

If `PromptAssembly.valid` is `False` (structural message validation failure),
falls back to legacy model messages. Fallback is rare.

## Agent coordination

### Dependency injection

All agents follow constructor injection. Unset optionals get defaults —
the coordinator self-assembles when dependencies are omitted.

```python
coordinator = CoordinatorAgent(
    database=db,
    memory_agent=ShortTermMemoryAgent(short_term_memory),
    context_builder=ContextBuilderAgent(short_term_memory),
    chat_agent=ChatAgent(ModelWrapper(model_name)),
    system_prompt="...",
    # All following optional with defaults:
    route_planner=RoutePlanner(),
    retriever_dispatcher=RetrieverDispatcher(db),
    memory_reranker=MemoryReranker(),
    context_budget_allocator=ContextBudgetAllocator(),
    trace_context_builder=TraceContextBuilder(),
    context_comparator=ContextComparator(),
    routing_agent=RoutingAgent(route_planner),
    context_manager_agent=ContextManagerAgent.for_model(model_id),
)
```

### Agent interaction summary

```text
CoordinatorAgent
  │
  ├──→ RoutingAgent.route(content) → RoutingDecision
  ├──→ require_document_memory() / require_chat_document_memory()
  ├──→ RetrieverDispatcher.retrieve(chat_id, route_plan) → MemoryCandidate[]
  ├──→ MemoryReranker.rank_with_trace(...) → ranked MemoryCandidate[]
  ├──→ RetrieverDispatcher.expand_document_neighbors(ranked) → expanded
  ├──→ _dump_retrieval_log(chat_id, turn_index, candidates, ...)
  ├──→ ContextManagerAgent.build_context_packet(...) → ContextManagerResult
  │      ├── [uses] ContextBudgetAllocator
  │      ├── [uses] EvidenceConstrainedContextSelector
  │      ├── [uses] DynamicWorkingMemoryBudgetPlanner
  │      └── [uses] ContextBuilder
  ├──→ ShortTermMemoryAgent.build_context(chat_id) → ShortTermContext
  ├──→ ContextBuilderAgent.build(...) → (messages, packet)
  ├──→ ContextComparator.compare(old, new, ...) → ContextComparison
  ├──→ ChatAgent.generate(messages) → answer
  ├──→ ShortTermMemoryAgent.update_memory_if_needed(chat_id) → bool
  └──→ [optional] LangGraph read-only shadow/demo orchestration
```

## DocumentIngestionAgent

Loads, indexes, and summarizes local files for document RAG:

```python
class DocumentIngestionAgent:
    def __init__(self, indexer=None, summary_model=None, summary_database=None)
    def index_file(path, display_name=None, *, document_id=None)
        → DocumentIngestionResult
    def _generate_summary(document_id, text) → None
```

### LLM document summarization (NEW)

After indexing, `_generate_summary()` sends first 8000 chars to LLM via
`SummaryModel` Protocol (minimal `chat()` interface). Result persisted via
`summary_database.update_document_summary()`. Best-effort — failures silently
ignored; indexing succeeds regardless. Uses `DOCUMENT_SUMMARY_SYSTEM_PROMPT`
(3-6 sentence, ≤300 word instruction).

Returns `DocumentIngestionResult(document_id, file_name, file_extension,
chunk_count, indexed, errors, metadata)`.

Falls back to `LangChainChromaRetriever.from_env()` when no indexer injected.

## Configuration

| Env variable | Default | Effect |
|---|---|---|
| `ORCHESTRATION_MODE` | `"native"` | `"native"`, `"langgraph_shadow"`, or `"langgraph_demo"`. |
| `ROUTING_MODE` | `"hybrid"` | `"rule"`, `"llm"`, or `"hybrid"`. |
| `CHAT_DOCUMENT_SCOPE_STICKY` | `"true"` | When true, chats with documents auto-enable document_memory. |
| `RETRIEVAL_LOG_ENABLED` | `"1"` | When true, dumps candidate ranking to JSON per turn. |
| `MODEL_NAME` | required | Model ID for `ChatAgent` and `ContextManagerAgent.for_model()`. |
| `CHAT_SYSTEM_PROMPT` | see `settings.py` | System prompt injected into `CoordinatorAgent`. |
| `MEMORY_UPDATE_POLICY` | `"scheduled"` | `"scheduled"`, `"agentic_each_turn"`, or `"chat_end_only"`. |
