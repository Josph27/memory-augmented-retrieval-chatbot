# src/documents/

**TL;DR:** Document ingestion pipeline — load `.txt`/`.md`/`.pdf` files, chunk
them with configurable splitters, resolve which documents each chat may retrieve,
and inspect Chroma metadata. All deterministic, no LLM calls in the core pipeline.
`DocumentIngestionAgent` (in `src/agents/`) optionally generates LLM-backed
document summaries at index-time.

## Architecture

Four-stage document-memory ingestion pipeline:

```text
load → chunk → embed/index → retrieve (chat-scoped)
```

Each stage is handled by a dedicated module with no cross-coupling:

| Module | Lines | Role | Key type |
|--------|-------|------|----------|
| `loaders.py` | 206 | File → plain text | `LoadedDocument` |
| `splitters.py` | 294 | Text → document chunks | `DocumentChunk`, `ChunkingConfig` |
| `registry.py` | 128 | Query → allowed document IDs | `DocumentResolution` |
| `inspection.py` | 148 | Chroma metadata → display rows | `DocumentInspectionRow` |

Embedding and indexing happen downstream in
`LangChainChromaRetriever.index_text_document()` (see
`src/retrieval/langchain_chroma_retriever.py`).

## Loading: `loaders.py` (206 lines)

```python
load_document_file(path, encoding="utf-8", display_name=None) -> LoadedDocument
```

Dispatches by file extension:

| Extension | Handler | Notes |
|-----------|---------|-------|
| `.txt`, `.md` | `load_text_file()` | UTF-8 read, no preprocessing |
| `.pdf` | `load_pdf_file()` | pypdf → PyMuPDF fallback |

**PDF handling details:**

- Max 512 pages (`MAX_PDF_PAGES`).
- Tries `load_pdf_with_pypdf()` first, falls back to `load_pdf_with_pymupdf()`.
  If both fail, raises `DocumentLoaderError`.
- `normalize_pdf_text()`: strips standalone page numbers (regex `\d{1,4}`),
  removes repeated header/footer lines (≥50% page threshold), collapses single
  `\n` to spaces, coalesces `\n` runs into `\n\n` paragraph breaks.
- Raises `DocumentLoaderError` for scanned/image PDFs with no extractable text.

**Key types:**

```python
@dataclass(frozen=True)
class LoadedDocument:
    title: str       # file stem (or display stem)
    text: str        # full normalized text
    source: str      # always "file"
    metadata: dict   # file_path, file_name, file_extension, loader_name, source
```

**Low-level loaders:**

```python
load_pdf_with_pypdf(path) -> tuple[str, int, str]   # (text, page_count, "pypdf")
load_pdf_with_pymupdf(path) -> tuple[str, int, str]  # (text, page_count, "pymupdf")
```

**Top-level conveniences:**

```python
index_loaded_document(loaded: LoadedDocument, indexer: TextDocumentIndexer)
# Index a pre-loaded document through a TextDocumentIndexer.

index_file_document(path, indexer, display_name=None)
# Load and index one local file. Wraps load_document_file + index_loaded_document.
```

`TextDocumentIndexer` is a Protocol with `index_text_document(title, text, source, metadata)`.
Implemented by `LangChainChromaRetriever`.

## Chunking: `splitters.py` (294 lines)

```python
split_document_text(text, config=None) -> list[DocumentChunk]
```

Two splitters, chosen via `DOCUMENT_CHUNKER` env var:

| Env value | Splitter | Description |
|-----------|----------|-------------|
| `custom` (default) | `CustomParagraphSplitter` | Paragraph-preserving; splits long paragraphs on sentence/word/hard boundaries |
| `langchain_recursive` | `LangChainRecursiveSplitter` | LangChain `RecursiveCharacterTextSplitter`; falls back to custom if import fails |

### Configuration

**`ChunkingConfig` dataclass:**

```python
@dataclass(frozen=True)
class ChunkingConfig:
    chunker: str          # "custom" or "langchain_recursive"
    chunk_size: int       # target chunk size (env override)
    chunk_overlap: int    # overlap for LangChain splitter
    target_chars: int     # custom splitter target (defaults to chunk_size)
    max_chars: int        # custom splitter max (max(chunk_size, provided_max))
```

`ChunkingConfig.from_env(target_chars, max_chars)` reads from env vars.

**Note on defaults:** The module-level constants (`DEFAULT_CHUNK_SIZE=256`,
`DEFAULT_CHUNK_OVERLAP=56`) are code-level fallbacks. At runtime, `settings.py`
overrides them via env vars to **1024/164**. The canonical values are in
`settings.py`.

### CustomParagraphSplitter

```python
split_text_custom(text, target_chars, max_chars) -> list[str]
```

Three-tier boundary fallback for splitting overlong paragraphs:

1. **Sentence** — `.`, `?`, `!` (via `best_split_index`)
2. **Word** — space
3. **Hard** — splits at `max_chars` if nothing found before it

Paragraphs are joined with `\n\n` separators. Accumulation tracks separator length
to stay under `target_chars`.

### LangChainRecursiveSplitter

Adapter with import resolution across two package layouts:

1. `langchain_text_splitters.RecursiveCharacterTextSplitter` (standalone package)
2. `langchain.text_splitter.RecursiveCharacterTextSplitter` (monolithic package)

If both fail, raises `LangChainSplitterUnavailable`. Uses `add_start_index=True`
for character-offset metadata. Overlap is clamped to `[0, chunk_size-1]`.

### Legacy helper

```python
split_text_into_chunks(text, target_chars=256, max_chars=256) -> list[str]
```

Returns plain string chunks via `CustomParagraphSplitter`. Kept for backward
compatibility. Uses code-level defaults (256), not runtime settings.

### Chunk metadata

Every `DocumentChunk` carries: `splitter_name`, `chunk_size`, `chunk_overlap`,
`fallback_used`, `start_char`, `end_char`. LangChain chunks also include
`source_chunk_index`. Custom chunks include `requested_splitter` when a
LangChain→custom fallback occurred.

## Document scope: `registry.py` (128 lines)

```python
class DocumentRegistry:
    def resolve(self, chat_id: str, query: str) -> DocumentResolution
```

Resolution strategy, in order:

1. **Chat-associated documents** — `database.documents_for_chat(chat_id)`.
2. **Global fallback** — when no chat-scoped documents exist, uses all ready
   documents (`list_all_documents(status="Ready")`).
3. **Explicit filename match** — word-boundary regex match against the query.
4. **Implicit reference** — 16 phrases detected (e.g. "this document",
   "the uploaded file", "summarize it"). Multiple matches → `DocumentAmbiguityError`.
   Documents still indexing → `DocumentNotReadyError`.
5. **Single document** — returns the only ready document.
6. **Multiple documents, no implicit reference** — returns all ready documents.

The resulting `DocumentResolution` feeds document IDs to the Chroma retriever
as a metadata filter, enforcing chat-scoped retrieval.

**Error hierarchy:**

```
DocumentScopeError (registry.py)
    ├─ DocumentAmbiguityError
    └─ DocumentNotReadyError
```

## Inspection: `inspection.py` (148 lines)

```python
chroma_document_inspection_rows(persist_dir, collection_name="documents_memory")
    -> list[DocumentInspectionRow]

rows_from_chroma_metadata(ids, metadatas) -> list[DocumentInspectionRow]

delete_document_chunks(persist_dir, document_id) -> int

format_document_inspection_rows(rows) -> str
```

Read-only access to Chroma collection metadata. Groups chunks by `document_id`,
extracts title/file/source/backend for the `/documents` UI page and CLI scripts.

Error: `DocumentInspectionUnavailable` (RuntimeError).

## Document summaries (not in this module)

`DocumentIngestionAgent` in `src/agents/document_ingestion_agent.py` generates
LLM-backed document summaries at index-time. It sends the first 8,000 characters
through a `SummaryModel.chat()` call with `DOCUMENT_SUMMARY_SYSTEM_PROMPT`.
Summaries are stored in the SQLite `documents` table (`summary_text` column)
via `Database.update_document_summary()`. Summary generation is best-effort —
failures are silently ignored.

## Environment variables

| Variable | Default | Module | Description |
|----------|---------|--------|-------------|
| `DOCUMENT_CHUNKER` | `custom` | splitters | `custom` or `langchain_recursive` |
| `DOCUMENT_CHUNK_SIZE` | `1024` | splitters | Target characters per chunk (runtime canonical) |
| `DOCUMENT_CHUNK_OVERLAP` | `164` | splitters | Chunk overlap (16% of chunk_size) |
| `DOCUMENT_TOP_K` | `18` | downstream | Post-reranker chunks in prompt |
| `DOCUMENT_RETRIEVAL_FETCH_LIMIT` | `42` | downstream | Chunks fetched from Chroma before reranking |
| `LANGCHAIN_CHUNK_SIZE` | `1024` | downstream | LangChain-specific chunk size for Chroma indexer |
| `LANGCHAIN_CHUNK_OVERLAP` | `164` | downstream | LangChain-specific chunk overlap for Chroma indexer |
| `EMBEDDING_MODEL_NAME` | `BAAI/bge-small-en-v1.5` | downstream | Embedding model for vector paths |

All canonical defaults live in `src/settings.py`. The splitters.py code-level
constants (256/56) are overridden at startup via `_setenv()` calls.

    └─ RuntimeError

    └─ DocumentNotReadyError

    └─ RuntimeError

    └─ RuntimeError
