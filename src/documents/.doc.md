# src/documents/

**TL;DR:** Document ingestion pipeline — load `.txt`/`.md`/`.pdf` files, chunk
them with configurable splitters, resolve which documents each chat may retrieve,
and inspect Chroma metadata. All deterministic, no LLM calls.

## Architecture

The module implements a four-stage document-memory ingestion pipeline:

```text
load → chunk → embed/index → retrieve (chat-scoped)
```

Each stage is handled by a dedicated module with no cross-coupling:

| Module | Role | Key type |
|--------|------|----------|
| `loaders.py` | File → plain text | `LoadedDocument` |
| `splitters.py` | Text → document chunks | `DocumentChunk` |
| `registry.py` | Query → allowed document IDs | `DocumentResolution` |
| `inspection.py` | Chroma metadata → display rows | `DocumentInspectionRow` |

Embedding and indexing happen downstream in
`LangChainChromaRetriever.index_text_document()` (see
`src/retrieval/langchain_chroma_retriever.py`).

## Loading: `loaders.py`

```python
load_document_file(path, encoding="utf-8", display_name=None) -> LoadedDocument
```

Dispatches by file extension:

| Extension | Handler | Notes |
|-----------|---------|-------|
| `.txt`, `.md` | `load_text_file()` | UTF-8 read, no preprocessing |
| `.pdf` | `load_pdf_file()` | Tries pypdf, falls back to PyMuPDF |

**PDF handling specifics:**

- Max 512 pages (`MAX_PDF_PAGES`).
- Normalization: strips standalone page numbers, removes repeated header/footer
  lines (≥50% of pages), collapses single newlines to spaces, coalesces
  multiple newlines into paragraph breaks.
- Raises `DocumentLoaderError` for scanned/image PDFs with no extractable text.

**Key types:**

```python
@dataclass(frozen=True)
class LoadedDocument:
    title: str       # file stem
    text: str        # full normalized text
    source: str      # always "file"
    metadata: dict   # file_path, file_name, file_extension, loader_name
```

**Top-level convenience:**

```python
index_file_document(path, indexer, display_name=None)
```

Loads the file and indexes it through the provided `TextDocumentIndexer`
(implemented by `LangChainChromaRetriever`).

## Chunking: `splitters.py`

```python
split_document_text(text, config=None) -> list[DocumentChunk]
```

Two splitters, chosen via `DOCUMENT_CHUNKER` env var:

| Env value | Splitter | Description |
|-----------|----------|-------------|
| `custom` (default) | `CustomParagraphSplitter` | Paragraph-preserving; splits long paragraphs on sentence boundaries |
| `langchain_recursive` | `LangChainRecursiveSplitter` | LangChain's `RecursiveCharacterTextSplitter`; falls back to custom if unavailable |

**Configuration** (all env vars):

| Variable | Default | Description |
|----------|---------|-------------|
| `DOCUMENT_CHUNKER` | `custom` | `custom` or `langchain_recursive` |
| `DOCUMENT_CHUNK_SIZE` | 1000 | Target characters per chunk |
| `DOCUMENT_CHUNK_OVERLAP` | 150 | Overlap between chunks (LangChain only) |

**Chunk metadata:** Every `DocumentChunk` carries `splitter_name`, `chunk_size`,
`chunk_overlap`, `fallback_used`, `start_char`, `end_char`.

**Legacy helper:**

```python
split_text_into_chunks(text, target_chars=800, max_chars=1000) -> list[str]
```

Returns plain string chunks. Kept for backward compatibility.

## Document scope: `registry.py`

```python
class DocumentRegistry:
    def resolve(self, chat_id: str, query: str) -> DocumentResolution
```

Resolution strategy, in order:

1. **Chat-associated documents** — queries `database.documents_for_chat(chat_id)`.
2. **Global fallback** — when no chat-scoped documents exist, uses all ready
   documents across every chat (`list_all_documents(status="Ready")`).
3. **Explicit filename match** — if the query contains a filename (word-boundary
   match), returns only that document.
4. **Implicit reference** — phrases like "this document", "the uploaded file",
   "summarize it" are detected. When multiple documents match, raises
   `DocumentAmbiguityError`. When documents are still indexing, raises
   `DocumentNotReadyError`.
5. **Single document** — returns the only ready document.
6. **Multiple documents, no implicit reference** — returns all ready documents.

The resulting `DocumentResolution` feeds the document IDs to the Chroma
retriever as a metadata filter, enforcing chat-scoped retrieval.

## Inspection: `inspection.py`

```python
chroma_document_inspection_rows(persist_dir, collection_name="documents_memory")
    -> list[DocumentInspectionRow]

rows_from_chroma_metadata(ids, metadatas) -> list[DocumentInspectionRow]

delete_document_chunks(persist_dir, document_id) -> int

format_document_inspection_rows(rows) -> str
```

Read-only access to Chroma collection metadata. Groups chunks by `document_id`,
extracts title/file/source/backend for the `/documents` UI page and CLI scripts.

## Environment variables

| Variable | Default | Module | Description |
|----------|---------|--------|-------------|
| `DOCUMENT_CHUNKER` | `custom` | splitters | Splitter selection |
| `DOCUMENT_CHUNK_SIZE` | 1000 | splitters | Target characters per chunk |
| `DOCUMENT_CHUNK_OVERLAP` | 150 | splitters | Chunk overlap |
| `EMBEDDING_MODEL_NAME` | — | downstream | Model used for embedding (in `langchain_chroma_retriever.py`) |

## Error hierarchy

```
DocumentLoaderError (loaders.py)
    └─ RuntimeError

DocumentScopeError (registry.py)
    ├─ DocumentAmbiguityError
    └─ DocumentNotReadyError

DocumentInspectionUnavailable (inspection.py)
    └─ RuntimeError
```
