# Retrieval (`src/retrieval/`)

**TL;DR:** 7 source retrievers → `MemoryCandidate[]` → `MemoryReranker` (11-feature deterministic + cross-encoder/hybrid/LLM modes) → ranked candidates. Neighbor expansion deferred post-reranking. Gists are orientation scaffolds; `GistRawSpanExpander` converts them into bounded exact SQLite evidence (`raw_message_span` candidates). Document summary detector returns pre-computed LLM summaries for overview queries. Two cross-encoder modes: pure CE (`--cross-encoder`) and per-source-normalized deterministic blend (`--hybrid`).

## Architecture

The retrieval subsystem implements a two-stage pipeline with post-reranking neighbor expansion:

```
RoutePlan.sources
    → RetrieverDispatcher (dispatch to source retrievers)
    → GistRawSpanExpander (gist → raw_span expansion)
    → MemoryReranker (deterministic + optional CE/hybrid/LLM)
    → RetrieverDispatcher.expand_document_neighbors() (post-reranking)
    → ranked MemoryCandidate[]
```

Key design: neighbor expansion runs **after** reranking so the cross-encoder sees clean single-chunk text. The dispatcher then fetches ±1 neighbors from the Chroma vectorstore and merges them inline into each candidate's content — producing fewer, richer candidates (3× context per chunk).

## Retriever contract

Every source retriever implements the `SourceRetriever` protocol:

```python
class SourceRetriever(Protocol):
    def retrieve(self, chat_id: str, source_plan: SourcePlan) -> list[MemoryCandidate]: ...
```

## `RetrieverDispatcher`

Central hub wiring 7 retrievers. `__init__` constructs defaults; any can be overridden for testing.

**`retrieve(chat_id, route_plan)`:**

1. Iterates `route_plan.sources`; skips disabled sources.
2. For `document_memory`, resolves document scope via `DocumentRegistry`. On `DocumentScopeError`, records error and continues.
3. Invokes matching retriever. Raises → recorded in `self.last_errors`, continues (fail-soft).
4. For `document_memory` queries returning empty despite allowed IDs, retries with fallback query (original query + document names).
5. For `context_profile=document_question` with no explicit limit, boosts limit to 40.
6. After all sources: runs `GistRawSpanExpander.expand()`.
7. Returns `candidates + expanded_gist_spans`.

**`expand_document_neighbors(candidates)`** — must be called post-reranking. Fetches ±1 neighbor chunks from the Chroma vectorstore and merges them inline into each document candidate's `content` field. Sets `sentence_window_expansion=True` and `neighbor_chunks_merged` counter. Non-document candidates pass through unchanged.

### `LangChainChromaRetriever`

470 lines (not 280). Key additions beyond basic vector retrieval:

**Summary query detection:** Checks query against `SUMMARY_QUERY_TERMS` (e.g., "summarize", "overview", "contents", "what are the contents"). When matched, calls `summary_getter` to retrieve a pre-computed LLM document summary. Summary candidate is inserted at position 0 with `skip_rerank=True`.

**Neighbor expansion:** `_expand_neighbors()` (module-level function) merges ±1 adjacent chunks into each candidate's content inline. Instead of creating separate low-score neighbor candidates, it produces richer 3×-context candidates. Called by `RetrieverDispatcher.expand_document_neighbors()` post-reranking.

**Document filtering:** `allowed_document_ids` in `source_plan.filters` scopes retrieval. Uses Chroma `$in` filter for multiple IDs.

**Construction defaults:**

- `persist_dir`: `data/chroma`
- `embedding_model_name`: `sentence-transformers/all-MiniLM-L6-v2` (class default; env overrides to `BAAI/bge-small-en-v1.5`)
- `chunk_size`: 1024, `chunk_overlap`: 164
- `default_top_k`: 4 (class), `fetch_limit`: 80
- `fallback_retriever` and `summary_getter` for dependency injection

**`from_env()` defaults:**

- `DOCUMENT_TOP_K`: 18 (from settings.py)
- `DOCUMENT_RETRIEVAL_FETCH_LIMIT`: 42

### `StructuredMemoryRetriever`

287 lines. Three modes via `STRUCTURED_MEMORY_RETRIEVAL_MODE`:

- **`sqlite`** (default): Queries `long_term_store` by namespace. Falls back to `chat_memory_state` (LangMem snapshot) + DB-active records not in snapshot.
- **`vector`**: Searches sqlite-vec long-term memory vector index; falls back to sqlite on `VectorIndexUnavailable` or any exception.
- **`hybrid`**: Combines sqlite and vector results, deduplicates by namespace + memory_id, returns up to `source_plan.limit` (default 10).

Note: The env default is `sqlite`, not `hybrid` as previously documented.

### `RawMessageSpanRetriever`

850 lines. Three retrieval modes:

1. **Explicit span mode**: triggered by `start_message_id`/`end_message_id` or `gist_id` in filters. Loads exact range. Supports `allowed_chat_ids` scope and anchor preservation.
2. **Direct mode**: cross-chat lexical search with density bonus scoring. Key parameters: `DIRECT_RAW_DEFAULT_LIMIT=12`, `DIRECT_RAW_WINDOW_MESSAGES=2`, `DIRECT_RAW_MAX_MESSAGES_PER_SPAN=12`, `LOCAL_QUERY_WINDOW_CHARS=320`. Scoring: 0.7×local_overlap + 0.3×len(overlap), density_bonus up to 0.04, uncommon term bonus up to 0.2.
3. **Global summary mode** (`context_profile=global_summary`): chronologically chunked raw messages from inactive chats using `chronological_message_chunks()`.

`source_plan_for_gist_candidate()` builds an explicit span source plan from gist candidate metadata — used by the coordinator when gist expansion is needed.

### `GistRawSpanExpander`

258 lines. Converts gist candidates into bounded `raw_message_span` candidates.

**Pipeline:**

```
Gist candidates (current_chat_gist / previous_chat_gist)
  → _request_for: resolve start/end message IDs from gist metadata or DB record
  → merge_requests: merge overlapping gist ranges (same chat_id, contiguous)
  → database.messages_for_chat_span: load raw messages from SQLite
  → bounded_messages: select max 12 messages around best lexical match
  → best_message_ids: identify anchor messages for char truncation
  → format_raw_span_with_anchor: anchor-aware formatting (max 4000 chars)
  → emit raw_message_span MemoryCandidates with full provenance
```

Key constants: `DEFAULT_GIST_EXPANSION_MAX_MESSAGES=12`, `DEFAULT_RAW_SPAN_MAX_CHARS=4000`.

Each expanded candidate carries: `parent_gist_id(s)`, `parent_source(s)`, `derived_from_source`, `original_start/end_message_id`, `anchor_message_ids`, `selection_reason`, `truncated` flag.

## MemoryReranker

1398 lines — the largest single file. Four modes controlled by `RERANKER_MODE`:

| Mode | Behavior |
|---|---|
| `deterministic` | 11 hand-crafted features, weighted sum, stable sort |
| `cross_encoder` | Deterministic pre-rank → raw CE scores (pure, no normalization or blend) |
| `hybrid` | Cross-encoder → per-source z-score normalization → deterministic blend → gated LLM rerank |
| `llm` | Deterministic scoring → listwise LLM reranking |

Current default: `cross_encoder` (see `settings.py:94`).

### `RerankerWeights` — 11 features

| Feature | Weight | What it measures |
|---|---|---|
| `lexical_overlap` | 0.35 | Query term coverage in candidate content (stopword-filtered) |
| `query_source_boost` | 0.25 | Source-specific boost when query explicitly references that source type |
| `semantic_score` | 0.20 | Vector/semantic relevance from candidate metadata |
| `similarity_score` | 0.20 | Similarity from metadata (default 0.5) |
| `importance` | 0.15 | Importance from metadata (default 0.5) |
| `confidence` | 0.15 | Confidence from metadata, score, or 0.5 default |
| `recency` | 0.10 | 1/(order+1) or metadata |
| `usage_count` | 0.05 | Normalized usage_count / 10 |
| `source_priority` | 0.10 | Static source priority from policy |
| `status_penalty` | 1.0 | Negative penalty: archived=0.15, superseded=0.7, deleted=1.0 |
| `redundancy_penalty` | 1.0 | Exact duplicate penalty = 0.08 per duplicate |

### Deterministic scoring

`MemoryReranker.score_candidate()` extracts 11 features via `candidate_features()`, computes weighted contributions, and produces `final_score`. Candidates with `skip_rerank=True` metadata preserve their original score. Sorting is `(-score, original_rank)` for stable tie-breaking. Every candidate receives `score_breakdown` metadata with features, weights, contributions, and final_score.

### Pure cross-encoder mode (`--cross-encoder` startup)

`_cross_encoder_rank()` — no per-source normalization or deterministic blend:

1. Runs cross-encoder on `CE_SCORABLE_SOURCES` only (`document_memory`, `structured_memory`).
2. `_sort_by_raw_ce_scores()`: CE-scored candidates ranked by raw CE score; non-CE sources (raw spans, gists, recent messages) preserve deterministic score, sorted after CE candidates.
3. Metadata: `normalized_cross_encoder_score=None`, `normalized_deterministic_score=None` (intentionally None in pure CE mode).

### Hybrid CE stage (`--hybrid` startup)

`_cross_encoder_stage()` — per-source z-score normalization + deterministic blend:

1. Runs cross-encoder on `CE_SCORABLE_SOURCES`.
2. `combine_cross_encoder_scores()`:
   - Per-source z-score normalization (via `_z_score_norm_list`): document_memory and structured_memory normalized independently to prevent one source dominating by scale alone.
   - Blends: `CE_weight × norm_ce + (1-CE_weight) × deterministic_score`.
   - Non-CE-scorable sources (`recent_messages`, `current_chat_span`, `raw_message_span`, gists) preserve original deterministic score with `cross_encoder_skipped=True`.
3. Single-candidate groups and identical values → 0.5 (neutral).

### LLM gate decision

`llm_gate_decision()` — only in hybrid mode (`hybrid_backend != "cross_encoder"`):

**Opens (use LLM) when:**

- **Provenance conflict**: top candidates include both `previous_chat_gist` and `raw_message_span` AND query asks for exact evidence.
- **Old decision conflict**: both `previous_chat_gist` and `raw_message_span` in top AND query asks about earlier decisions.
- **Cross-source ambiguity**: multiple sources in top candidates AND (margin ≤ ambiguity_margin OR rankings_disagree).

**Stays closed when:**

- ≤1 candidate.
- All top candidates share the same source (when `require_cross_source_conflict=True`).
- Top margin > `ambiguity_margin` and rankings agree.

`is_provenance_query()` checks for terms like "exactly", "quote", "evidence", "what did i say". `is_old_decision_query()` checks for "previous decision", "what did we decide", "earlier chat".

### LLM rerank stage

Listwise LLM reranking with JSON structured output. Validates candidate IDs, deduplication, numeric confidence. Falls back to previous stage when: model missing, confidence < `llm_min_confidence` (0.55), or any exception.

## Cross-encoder backend (`cross_encoder_reranker.py`)

102 lines. `SentenceTransformersCrossEncoderBackend`:

- Lazy-loads model on first `score()` call (or via `preload()`).
- Sigmoid-normalizes logit scores to [0,1] when values fall outside that range.
- `CrossEncoderBackend` protocol for dependency injection.
- `CrossEncoderUnavailable` for missing dependencies.
- `DEFAULT_CROSS_ENCODER_MODEL = "cross-encoder/ms-marco-MiniLM-L12-v2"`.

## Error handling

- **Per-source fail-soft:** `RetrieverDispatcher` catches exceptions, records in `self.last_errors`, continues.
- **Document scope errors:** `DocumentScopeError` caught before retriever invocation.
- **Gist expansion is best-effort:** empty spans silently skipped.
- **Reranker fallbacks:** CE errors → deterministic. LLM errors → previous stage. All recorded in trace metadata.
