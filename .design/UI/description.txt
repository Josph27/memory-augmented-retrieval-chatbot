## Implementation progression proposal:
    ### Agentic Loops:
        1. Planning loop:
        ... Planner -> Oracle -> Planner -> Oracle ...
            - Until the plan has no obvious flaws, if oracle uncovers some problems that pose architecture decisions, ask user for feedback before continuint the loop.
        2. Implementation loop:
        ... (... Worker -> Reviewer -> Worker -> Reviewer ...) -> Oracle ...
            - Worker / reviewer implement the plan, when the implementation is finished, oracle assess the situation and compares to original plans, defines required reworks, decides whether another planning loop is required or whether the fixes are simple enough to be fixed in a single pass. Repeats evaluation until there are no problems.
